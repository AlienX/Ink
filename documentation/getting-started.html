Introduction

Ink was made, as any framework, to relieve web developers and designers, from the tedious everyday tasks of handling diferent browsers and their particular quirks, provide commonly used UI elements and provide a simple default look and feel that can easily adapt to any screen size.

{LESS}

We chose {LESS} as the tool to help making the task of writting all the css a lot easier, and in the process have created a ton of helpful mixins that we use regularly, and hopefully will also help you getting your projects done a little easier.

Ink is highly customizable. You can change base colors, dimensions and even class names, however to tap into all this you'll need {LESS}, so we strongly encourage you to check out the {LESS} site and documentation.

The grid

We took into consideration, existing browsers css rendering capabilities and the fact that we felt that it's a lot more intuitive to divide space in percentages. Rather than having to count abstract base grid column units to get, let's say a 1/3 width column, you can simply say that you need a 33% width column. This also enabled us to come up with a completely fluid, screen size aware, grid system.

We also chose to use the "new" border-box box model to build our grid system. This helps to keep layouts flexible with less code and allows us to get optional gutters between grid divisions without the need for aditional markup.

To achieve this we've set 3 default screen sizes: large (for screens wider than 960px), medium (for screen widths between 960px and 650px) and small (for screen widths below 650px).

In order to make Ink compatible with as many browsers as possible we've taken the following approach:

  • Modern browsers that can handle media queries and the border-box box model get the full fluid layouts and media query goodness.
  • Older Browsers (IE8) that can't handle media queries only get the grid for large displays. This is the reason why the large breakpoint is not wrapped in an media query.
  • Even older browsers (IE7) that can't handle media queries or the border-box box model, get a fixed widths grid in a specific css file. IE conditional comments are required to use it.

First steps

Download Ink

Download a fresh copy of Inks latest version, you'll get a zip package containing all kinds of treats.

 ink/
  ├── cookbook/ (recipes for common layout types using Ink)
  ├── css/
  │   ├── ink-ie7-min.js (minified css for IE7)
  │   ├── ink-ie7.js (css for IE7)
  │   ├── ink-min.css (main css, minified)
  │   ├── ink.css (main css)
  │   └── your.css (a simple template to help you getting started)
  ├── documentation/ (you guessed it. It's a copy of this site)
  ├── font/ (font files for Ubuntu fonts and FontAwesome)
  ├── js/ (Ink.js and some documentation helper JS files are in here)
  │   ├── ink.js (Inks core js library)
  │   ├── …
  │   └── ink.progressbar.js (progress bar js pluging)
  └── less/
      ├── config/ (configuration files for Ink modules)
      │   ├── __alerts.less
      │   ├── …
      │   └── __widgets.less
      ├── modules/ (Ink modules styling)
      │   ├── alerts.less
      │   ├── …
      │   └── utilities.less
      ├── ink-ie.less (generates ink-ie7.css)
      ├── ink.less (@imports all modules, config files and generates ink.css)
      └── site.less (generates css for the documentation)
