Form building

There are some essential classes you'll use to build forms with Ink: ink-form, control-group and control.

To get started, add the ink-form class to your form element.

To add labels, form fields and tips use a div element with the control-group class to wrap each label / field pair. The form field should then be wrapped in a div with the control class. The <label> for each field is located above the field, in block.

This will render a form like the one shown below.

code

<form class="ink-form">
    <fieldset>
        <div class="control-group">
            <label for="text-input">Text input</label> 
            <div class="control">
                <input id="text-input" type="text" placeholder="Please input some text">
            </div>
        </div>
    </fieldset>
</form>

Input file field

A small exception goes to the <input type="file"> field, were an adicional <div> wrapper with input-file class is needed to achieve a consistent look across browers.

code

<div class="control-group">
    <label>File input</label> 
    <div class="control">
        <div class="input-file"><input type="file"></div>
    </div>
</div>

Adding tips

Adding tips to forms fields is easy. Just add the message by using a paragraph element with the tip class.

You can add help text to fields

code

<div class="control-group">
    <label for="text-input">Text input</label> 
    <div class="control">
        <input id="text-input" type="text" placeholder="Please input some text">
    </div>
    <p class="tip">You can add help text to fields</p>
</div>

Block form with inline fields

Here's an example of block type forms used inline.

code

<form action="#" class="ink-form">
    <fieldset class="column-group gutters">
        <div class="control-group large-33 medium-33 small-100">
            <label for="name">Name</label>
            <div class="control">
                <input type="text" id="name">
            </div>
        </div>
        <div class="control-group large-33 medium-33 small-100">
            <label for="phone">Phone</label>
            <div class="control">
                <input type="text" id="phone">
            </div>
        </div>
        <div class="control-group large-33 medium-33 small-100">
            <label for="email">Email</label>
            <div class="control">
                <input type="text" id="email">
            </div>
        </div>
    </fieldset>
</form>

Inline form

Inline forms follow the same structure as the block ones.

Only here we take advantaje of Ink's grid system and add the column-group class to the control-group wrapper, after that you just have to define the width of the label and control using the proper grid classes like shown in the code example.

You can add help text to fields

code

<form class="ink-form">
    <fieldset>
        <div class="control-group column-group gutters">
            <label class="large-20 content-right" for="phone">Text input</label> 
            <div class="control large-80">
                <input type="text" id="phone">
            </div>
        </div>
    </fieldset>
</form>

Required fields and warnings

Ink forms also have modifier classes to address and highlight warnings and required fields. So if you have a required field, use the field wrapping element control-group to add a required class.

You can add help text to fields

code

<form action="#" class="ink-form">
    <fieldset class="column-group gutters">
        <div class="control-group required validation error">
            <label for="text-input">Text input</label> 
            <div class="control">
                <input id="text-input" type="text" placeholder="Please input some text">
            </div>
            <p class="tip">You can add help text to fields</p>
        </div>
    </fieldset>
</form>

If you need to print a warning or an error message near your field, then add the validation class to the wrapper control-group and a warning or error class. Also add a message using a paragraph element with the tip.

Warn about something

This field is required

code

<form action="#" class="ink-form">
    <fieldset class="column-group gutters">
        <div class="control-group required validation warning">
            <label for="text-input">Text input</label> 
            <div class="control">
                <input id="text-input" type="text" placeholder="Please input some text">
            </div>
            <p class="tip">Warn about something</p>
        </div>
        <div class="control-group required validation error">
            <label for="text-input">Text input</label> 
            <div class="control">
                <input id="text-input" type="text" placeholder="Please input some text">
            </div>
            <p class="tip">This field is required</p>
        </div>
    </fieldset>
</form>

Checkboxes and radio buttons

Lists of checkboxes or radio buttons follow the same structure of the previous examples, only here your inputs should be wrapped by an <ul> with the same control class used before.

If you need to add a label to the field group, add a <p> element with the label class. This pseudo-label will also display an icon if the control group contains required fields.

Group of checkboxes

Please select one or more options

Note: to remove any undesired paddings or list-style add the class unstyled to the <ul>, like shown in the lists section.

code

<form action="" class="ink-form">
    <fieldset>
        <legend>Group of checkboxes</legend>
        <div class="control-group">
            <p class="label">Please select one or more options</p>
            <ul class="control unstyled">
                <li>
                    <input type="checkbox" id="cb1">
                    <label for="cb1">Option 1</label>
                </li>
                ...
            </ul>
        </div>
    </fieldset>                     
</form>
Group of radio buttons

code

<form action="" class="ink-form">

                    <input type="radio" id="rb1">

Buttons

Your forms will need at least one button. You can crate a button with one of several markup elements, although button is most appropriate. Just add the ink-button class for the button layout and then use a utility class to give it a specific meaning. Available styling is shown below.

type active state disabled state code
Info 
<button class="ink-button info">Info</button>
<button class="ink-button info" disabled>Info</button>
Default 
<button class="ink-button">Default</button>
<button class="ink-button" disabled>Default</button>
Success 
<button class="ink-button success">Success</button>
<button class="ink-button success" disabled>Success</button>
Warning 
<button class="ink-button warning">Warning</button>
<button class="ink-button warning" disabled>Warning</button>
Caution 
<button class="ink-button caution">Caution</button>
<button class="ink-button caution" disabled>Caution</button>

Button groups

Need to group buttons? Ink has an easy solution for that. Wrap your ink-buttons in a div with the button-group class.

code

<div class="button-group">
    <button class="ink-button">left</button>
    <button class="ink-button">middle</button>
    <button class="ink-button">right</button>
</div>

Inline button groups

To achieve a button toolbar look wrap your button groups in a div with the button-toolbar class.

code

<div class="button-toolbar">
    <div class="button-group">
        <button class="ink-button"><span class="icon-file"></span></button>
        <button class="ink-button"><span class="icon-save"></span></button>
    </div>

    <div class="button-group">
        <button class="ink-button"><span class="icon-bold"></span></button>
        <button class="ink-button"><span class="icon-italic"></span></button>
        <button class="ink-button"><span class="icon-underline"></span></button>
    </div>

    <div class="button-group">
        <button class="ink-button"><span class="icon-align-left"></span></button>
        <button class="ink-button"><span class="icon-align-center"></span></button>
        <button class="ink-button"><span class="icon-align-right"></span></button>
        <button class="ink-button"><span class="icon-align-justify"></span></button>
    </div>
</div>
